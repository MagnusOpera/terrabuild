Welcome to Terrabuild! This section will guide you through understanding and using Terrabuild to build your monorepo.

## Your Learning Journey

Terrabuild is built around a few core concepts that work together:

1. **The Build Graph** - How Terrabuild structures your projects and dependencies
2. **Caching** - How Terrabuild makes builds fast by reusing unchanged work
3. **Tasks** - How Terrabuild executes builds using the graph and cache

Understanding these concepts will help you configure Terrabuild effectively and troubleshoot issues when they arise.

## Where to Start

**New to Terrabuild?** Start with the [Graph](/docs/getting-started/graph) to understand how Terrabuild structures builds, then follow the natural progression through [Caching](/docs/getting-started/caching) and [Tasks](/docs/getting-started/tasks).

**Ready to build?** Jump to [Install](/docs/getting-started/install) to get Terrabuild set up, then try the [Quick Start](/docs/getting-started/quick-start) tutorial with a working example.

**Setting up a monorepo?** Use the [Scaffolding](/docs/getting-started/scaffolding) tool to automatically generate configuration files.

## What You'll Learn

As you progress through this section, you'll understand:

- How Terrabuild discovers projects and builds a dependency graph
- How caching works and why it makes builds fast
- How tasks execute and when they build vs restore
- How to configure your workspace and projects
- How to optimize builds with batch operations

By the end, you'll have a solid understanding of how Terrabuild works and how to use it effectively in your projects.
